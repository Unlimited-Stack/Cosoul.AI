# i18n 多语种与实时翻译方案

> 适用于 Cosoul.AI 全平台（Web + Native），覆盖系统 UI 静态翻译和 UGC 内容实时翻译两条路线。

---

## 一、整体策略：双轨并行

| 场景 | 方案 | 说明 |
|------|------|------|
| **系统 UI**（按钮、导航、提示语等） | 传统 i18n 静态翻译文件 | 成本低、零延迟、可离线 |
| **UGC 内容**（帖子、聊天、Agent 对话） | LLM 实时翻译 + 缓存 | 自然流畅、上下文感知、无翻译腔 |

---

## 二、系统 UI — 静态 i18n

### 2.1 技术选型

- **i18next** + **react-i18next**：Web 和 Native 共用同一套 API
- **expo-localization**（Native）：读取系统语言
- **navigator.language**（Web）：读取浏览器语言

### 2.2 包结构（Turborepo 规范）

```
packages/i18n/
  src/
    locales/
      zh-CN.json        # 中文（默认）
      en-US.json        # 英文
      ja-JP.json        # 日语（按需扩展）
    index.ts            # i18next 初始化 + useTranslation 导出
  package.json
```

- `apps/` 只调用初始化，不写翻译逻辑（薄壳原则）
- 所有翻译 key 统一管理在 `packages/i18n`

### 2.3 语言优先级

```
用户手动选择 > 系统/浏览器语言 > 默认 zh-CN
```

- Web 持久化：`localStorage`
- Native 持久化：`AsyncStorage`

### 2.4 Web 端语言切换器

浏览器不支持系统级强制切换语言，需内置切换入口：
- 放在 Sidebar 底部（设置齿轮旁）或 Settings 页面内
- 切换后立即生效，无需刷新

### 2.5 艺术字体 / 特殊 Unicode 字符

| 类型 | 处理方式 |
|------|----------|
| 纯装饰性文字 | 作为图片/SVG 资源，每个语种一套 |
| 品牌 slogan | 在翻译 JSON 中直接写对应语种版本，不走自动翻译 |
| UGC 中的特殊字符 | 保持原样，不翻译 |

## 三、UGC 内容 — LLM 实时翻译

### 3.1 核心原理（Reddit 方案解析）

传统 NMT（Google Translate / DeepL）逐句翻译，丢失上下文，英译中"翻译腔"严重。

LLM 翻译的本质区别：
- **上下文窗口**：标题 + 正文 + 评论链一起送入，模型理解完整语境后翻译
- **风格指令**：prompt 指定"自然口语化，保留梗的意译"，读起来像母语者写的
- **跨语种隐身**：接收方完全感知不到对方使用的是什么语言

### 3.2 架构流程

```
用户请求内容 → API Gateway
  │
  ├─ 命中缓存（Redis, key = content_id + target_lang）→ 直接返回
  │
  └─ 未命中 ↓
     拼装 Prompt:
       system: "专业本地化译者，目标: {lang}，风格: 社区口语化，保留原味意译"
       user: { title, body, comments[] }
     │
     ↓ 调用 LLM API（流式）
     │
     ↓ 写入缓存（TTL 按内容热度动态调整）
     │
     └─ 返回翻译结果
```

### 3.3 关键工程问题

| 问题 | 解法 |
|------|------|
| **成本控制** | 热门内容翻译一次缓存复用；冷门内容按需翻译；使用性价比高的模型 |
| **延迟优化** | 流式返回 + 预翻译热门内容 + CDN 边缘缓存 |
| **术语一致** | 同一帖子的标题/正文/评论用同一次请求翻译 |
| **俚语与梗** | LLM 天然理解 meme，prompt 中加"保留原味意译"指令 |
| **语种检测** | 发送时标记源语言；接收方语言不同时自动触发翻译 |

### 3.4 与现有架构的结合

项目已有 `@repo/core/llm` 基础设施，翻译能力可作为轻量 Agent 内置：

```
packages/core/src/llm/
  translate.ts          # 翻译 service：detectLang() + translateContent()
```

- Web：经 BFF 转发（`/api/llm/translate`）
- Native：直连 Coding Plan
- 聊天场景：接收方客户端检测语种不同 → 自动调用翻译 → 无缝展示

## 四、实施路线

| 阶段 | 内容 | 优先级 |
|------|------|--------|
| **P0** | 搭建 `packages/i18n`，系统 UI 中英文翻译 | 高 |
| **P1** | Web 端语言切换器 + Native 跟随系统语言 | 高 |
| **P2** | LLM 翻译 service + Redis 缓存层 | 中 |
| **P3** | 聊天消息实时翻译 + 翻译 Agent 集成 | 中 |
| **P4** | 更多语种扩展（日/韩/西/法等） | 低 |
